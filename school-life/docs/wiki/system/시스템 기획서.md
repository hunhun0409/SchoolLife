# School Life Loop - 시스템 기획서 v1.0

## 1. 게임 전체상

### 장르
- 학교 생활 성장 시뮬레이션
- 주간 스케줄 관리
- 관계 기반 이벤트
- 미니게임
- 회차 반복 구조
- 추리 요소

### 핵심 구조
플레이어는 3학년 2학기를 반복하며 스탯, 관계, 단서를 쌓고 최종적으로 루프를 탈출한다.

게임은 크게 아래 흐름으로 진행된다.

1. 평일 스케줄 선택
2. 요일별 결과 처리
3. 특수 이벤트 발생 시 대화/이벤트 전환
4. 주말 행동 선택
5. 새로운 만남, 아르바이트, 휴식, 이벤트 처리
6. 정기 이벤트 처리
7. 학기 종료
8. 조건 미달 시 루프 반복
9. 점수 정산
10. 특전 구매
11. 다음 회차 시작

---

## 2. 전체 기간

### 플레이 기간
- 3학년 2학기
- 9월부터 12월 또는 1월까지
- 약 16주 기준

### 시간 단위
- 1턴 = 1일
- 1주 = 7일
- 월요일부터 금요일까지는 평일 성장 행동
- 토요일과 일요일은 주말 특수 행동

---

## 3. 평일 스케줄 시스템

### 기본 구조
평일에는 달력 UI가 표시된다. 플레이어는 월요일부터 금요일까지 순서대로 각 요일의 집중 행동을 선택한다.

예시 흐름:

1. 월요일 행동 선택
2. 실행
3. 결과 표시
4. 화요일 행동 선택
5. 실행
6. 결과 표시
7. 금요일까지 반복

특정 요일에 특수 이벤트가 발생하면 자동 진행을 멈추고 `DialogueScene` 또는 `EventScene`으로 전환한다.

### 평일 행동 카테고리
- 학업
- 운동
- 매력 활동

### 행동 예시

#### 학업
- 문제 풀이
- 복습
- 과제
- 예습

#### 운동
- 달리기
- 근력 운동
- 체력 훈련
- 스트레칭

#### 매력 활동
- 외모 관리
- 말하기 연습
- 공연 연습
- 학교 활동

### 강도 선택
각 행동은 강도를 선택할 수 있다.

| 강도 | 성장 배율 | 피로도 증가 |
| --- | ---: | ---: |
| 집중 | 1.5 | +5 |
| 보통 | 1.0 | +3 |
| 가볍게 | 0.5 | +1 |

### 결과 계산 기본식
```text
스탯 증가 = 행동 기본값 x 강도 배율 x 성장 보정 - 피로도 페널티
```

### 설계 의도
- 평일은 빠르게 진행한다.
- 반복 피로를 줄인다.
- 매일 세부 조작보다 주간 관리 경험을 강화한다.
- 특수 이벤트가 있을 때만 흐름을 끊고 스토리로 전환한다.

## 4. 주말 시스템

### 기본 구조
주말은 게임의 특수 플레이 구간이다. 토요일과 일요일에는 각각 기본 2회 행동할 수 있다. 특전 해금 후에는 하루 3회 행동도 가능하다.

### 주말 행동 흐름
1. 장소 선택
2. NPC 조우 판정
3. 행동 선택
4. 결과 처리
5. 친밀도, 스탯, 단서 반영

### 주말 행동 종류
- 히로인 만나기
- 아르바이트
- 휴식
- 특수 이벤트
- 연속 이벤트
- 장소 기반 조우

### 장소 예시

| 장소 | 주요 성장 인물 | 주요 행동 |
| --- | --- | --- |
| 도서관 | 모범생 히로인 | 같이 공부, 자료 조사 |
| 운동부 | 운동부 히로인 | 같이 운동, 대회 준비 |
| 카페 / 번화가 | 인기 많은 히로인 | 대화 이벤트, 휴식 |
| 학교 | 공통 이벤트 | 특수 이벤트, 연속 이벤트 |

히로인과 장소에 따라 성장 확률이 달라진다.

### 연속 이벤트
주말 만남에서 특정 조건을 만족하면 다음 주말에 연속 이벤트가 열릴 수 있다.

연속 이벤트가 있으면 다음 보너스를 제공한다.

- 해당 히로인 성장 100%
- 특정 이벤트 발생 가능
- 친밀도 보너스 가능

## 5. 스탯 시스템

### 기본 스탯
- 지성
- 체력
- 매력
- 피로도

### 스탯별 역할

#### 지성
- 시험 점수에 영향
- 공부 효율에 영향
- 일부 단서 해석 조건에 영향

#### 체력
- 운동 결과에 영향
- 체육대회 결과에 영향
- 피로도 관리에 영향
- 장기 시험 집중력에 영향

#### 매력
- 관계 보너스와 축제 결과에 영향
- 대화 및 이벤트 성공률에 영향

#### 피로도
- 행동 효율을 감소시킨다.
- 특정 이벤트 실패 조건이 될 수 있다.
- 휴식 필요성을 만드는 제한 수치다.

### 보조 효과
- 아이템 구매
- 피로도 회복
- 이벤트 선택지 해금

## 6. 미니게임 시스템

### 기본 방향
미니게임은 모든 행동에 항상 들어가지 않는다. 주로 중요한 행동, 정기 이벤트, 특수 이벤트에서 사용한다.

### 미니게임 종류

#### 공부 미니게임
- 퀴즈
- 간단한 계산
- 제한 시간 안에 정답 선택

스탯 영향:
- 지성이 높을수록 제한 시간 증가
- 힌트 제공 가능
- 오답 페널티 감소 가능

#### 운동 미니게임
- 좌우 입력 리듬
- 타이밍 입력
- 샌드백 치기 계열

스탯 영향:
- 체력이 높을수록 판정 완화
- 콤보 유지 쉬움
- 실수 시 손실 감소

#### 매력 미니게임
- 리듬 또는 타이밍
- 공연 연습
- 발표, 노래, 연극 계열

스탯 영향:
- 매력이 높을수록 점수 배수 증가
- 관객 반응 보너스
- 실패 시 페널티 감소

---

## 7. 관계 시스템

### 기본 요소
- 히로인별 친밀도
- 이벤트 단계
- 연속 상태
- 협력 보너스
- 친밀도 단계

| 단계 | 요구 친밀도 | 역할 |
| --- | ---: | --- |
| 1단계 | 20 | 기본 이벤트 |
| 2단계 | 40 | 관계 강화 |
| 3단계 | 60 | 단서 해금 |
| 4단계 | 80 | 특수 조건 |

### 관계 보너스
관계는 단순 엔딩 조건이 아니라 성능 증폭 장치다.

예시:
- 모범생 히로인과 친밀하면 공부 효율 증가
- 운동부 히로인과 친밀하면 체력 훈련 효율 증가
- 인기 많은 히로인과 친밀하면 매력 상승과 친밀도 상승 보정

## 8. 히로인 이벤트 시스템

히로인 이벤트는 단계형으로 진행한다.

| 조건 | 이벤트 |
| --- | --- |
| 친밀도 20 | 이벤트 1 |
| 친밀도 40 | 이벤트 2 |
| 친밀도 60 | 이벤트 3 |
| 친밀도 80 | 이벤트 4 |

### 특징
- 모든 히로인 루트는 동시에 진행 가능하다.
- 연계 엔딩이 가능하다.
- 특정 이벤트는 단서 획득 조건을 포함한다.
- 특정 이벤트는 루프 탈출 조건을 포함한다.

## 9. 단서 시스템

### 기본 방향
추리는 가볍게 간다. 플레이어가 단서를 직접 조합하지 않아도 조건을 충족하면 자동으로 정리된다.

### 단서 개수
- 총 10~15개
- 초기 기준 12개 권장

### 단서 분류
- 시간 관련 단서
- 캐릭터 관련 단서
- 사건 관련 단서

### 획득 방식
- 이벤트 기반
- 선택 기반
- 히로인 친밀도 기반
- 정기 이벤트 결과 기반

### 자동 조합
특정 단서가 모이면 시스템이 자동으로 히로인 결론을 해금한다.

예시:
- 단서 A: 반복되는 날짜
- 단서 B: 기억의 일부를 유지하는 인물
- 단서 C: 특정 히로인의 이상한 발언

자동 조합 결과:

```text
3학년 2학기가 반복되고 있다.
```

## 10. 정기 이벤트 시스템

### 고정 정기 이벤트
- 중간고사
- 체육대회
- 축제
- 기말고사

### 배치 예시
| 시기 | 이벤트 |
| --- | --- |
| 9월 | 중간고사 |
| 10월 | 체육대회 |
| 11월 | 축제 |
| 12월 | 기말고사 |

### 설계 원칙
모든 정기 이벤트는 지성, 체력, 매력을 모두 사용한다.

#### 시험
- 지성 중심
- 체력은 집중력과 장기 시험 유지에 영향
- 매력은 그룹 과제, 발표, 선생님 평가 보조 요소

#### 체육대회
- 체력 중심
- 매력은 팀워크와 응원에 영향
- 지성은 전략 선택에 영향

#### 축제
- 매력 중심
- 지성은 준비 완성도에 영향
- 체력은 당일 컨디션에 영향
- 관계는 이벤트 성공률에 영향

## 11. 루프 시스템

### 기본 구조
학기 종료 시 탈출 조건을 확인한다.

조건을 만족하지 못하면 다음 흐름으로 루프를 반복한다.

1. 점수 정산
2. 사인점 지급
3. 특전 구매
4. 다음 회차 시작

### 루프 시 초기화
- 날짜
- 스탯
- 이벤트 진행 상태
- 친밀도 일부 또는 전체

### 루프 시 유지
- 단서
- 특전
- 시스템 해금
- 일부 기억 관련 연출

### 플레이어 지식 목표
- 1회차: 대부분 실패
- 2회차: 루프 구조 이해
- 3회차: 탈출 가능성 확인
- 4회차: 완전 플레이 또는 진엔딩 가능

---

## 12. 특전 시스템

### 사인점 획득
학기 종료 시 평가 점수에 따라 사인점을 획득한다.

### 특전 예시
- 초기 지성 증가
- 초기 체력 증가
- 초기 매력 증가
- 초기 자금 증가
- 친밀도 상승률 증가
- 피로도 증가량 감소
- 평일 성장량 증가
- 주말 행동 횟수 +1
- 특정 아이템 보유 시작

### QoL 특전
회차 피로를 줄이기 위한 특전이다.

- 평일 결과 배속
- 이벤트 스킵
- 이미 본 이벤트 자동 스킵
- 주간 자동 스케줄 저장

## 13. 스킬 시스템

### 기본 방향
스킬은 1회차 이후 성장 요소로 획득한다.

### 획득 방식
지성, 체력, 매력 레벨마다 테마별 스킬 사인점을 얻는다.

### 사용 타이밍
하루 또는 주 종료 시 스킬 선택 화면에서 사용한다.

### 스킬 방향
- 학업 효율 증가
- 체력 훈련 효율 증가
- 매력 활동 효율 증가
- 피로도 감소
- 친밀도 상승 보정
- 미니게임 보정

세부 밸런스는 추후 진행한다.

## 14. 엔딩 시스템

### 엔딩 종류
- 배드 엔딩
- 히로인 엔딩
- 하렘 엔딩
- 루프 탈출 엔딩
- 진엔딩

### 배드 엔딩
루프 탈출 조건을 만족하지 못하고 기본 학기를 종료한다.

### 히로인 엔딩
특정 히로인의 친밀도와 이벤트 조건을 만족한다.

### 하렘 엔딩
히로인 3명 친밀도 80 이상을 만족한다.

### 루프 탈출 엔딩
필수 단서와 히로인 이벤트 조건을 만족하고 최종 이벤트에 성공한다.

### 진엔딩
아래 조건을 모두 만족한다.

- 회차 1회 이상
- 체육대회 1회 이상 완료
- 루프 탈출 조건 충족
- 히로인 관계 조건 충족

## 17. 구현 순서

### 1단계: 핵심 루프
